### [IN-PROGRESS] Feature: Wendao Common Fuzzy Search Primitives

The repo-overview / repo-sync baseline cleanup is now also landed: analysis orchestration again merges plugin-provided repository metadata, relation enrichment, and skeptic verification output, while managed mirrors/checkouts now preserve read-only `status` semantics, refresh bare-mirror branch heads correctly, and emit drift/freshness metadata that makes the filtered `repo_overview` and `repo_sync` `xiuxian-testing-gate` slices green again. The fuzzy hot path itself now uses a three-row scratch-buffer edit-distance implementation, identifier-aware Tantivy fragment splitting for CamelCase / alpha-digit names, matcher-local char/scratch reuse across candidate loops, thread-local buffer reuse for the public scoring helpers, borrowed-slice Tantivy fragment iteration so the per-document `Vec<String>` fragment materialization is gone from the common rescoring path, a single-pass identifier-boundary state machine instead of `char_indices().collect::<Vec<_>>()`, thread-local buffer reuse inside `LexicalMatcher::search` itself so the shared lexical search loop no longer creates per-call `query_chars` / `candidate_chars` / `scratch` vectors, and now also a single-pass Unicode-aware prefix gate so each candidate is lowercased exactly once while the leading-character requirement is checked with `char::to_lowercase()` equality instead of ASCII-only case folding; both the boundary splitter, the consecutive lexical-search buffer clearing, and the non-ASCII prefix gate now have direct unit coverage. `FuzzySearchOptions` now also exposes a preparatory `camel_case_symbol()` profile for future Julia/Modelica CamelCase-abbreviation tuning. `TopologyIndex` keeps the shared typo-tolerant title fallback, Studio definition resolution keeps the shared lexical fallback, and `fuzzy_suggest` plus graph dedup now reuse the same edit-distance helpers.
