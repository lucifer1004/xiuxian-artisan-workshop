# GTD: Sovereign Nexus (2026-04-09)

## 1. [ACTIVE] Wendao DuckDB Bounded Analytics

- **Task**: Audit the residual architectural value of DataFusion after the
  DuckDB-first search execution direction was clarified.
- **Status**: [Residual Value Audit Landed] The current code and docs are now
  tightened around one narrower split: `WendaoArrow`, pyarrow, julia-arrow,
  and Flight own the cross-language Arrow `RecordBatch` substrate; DuckDB is
  the intended search-side database execution lane over Parquet and bounded
  request-scoped analytic relations; DataFusion retains only residual Rust-side
  live Arrow compute and migration-baseline value where the data is still a
  generated Arrow workset.

## 2. [ACTIVE] Rust AST Flight Cutover

- **Task**: Remove the remaining Rust-local Julia and Modelica AST ownership
  from the active Wendao gateway path and route Studio `code_search` through
  the Julia-owned native parser line.
- **Status**: [Gateway Native Route Audit] `xiuxian-wendao-modelica/` is
  retired from the active Rust ownership line, `xiuxian-ast` no longer exports
  Julia or Modelica tree-sitter surfaces, and Studio `intent = "code_search"`
  now runs through a source-owned gateway handler plus shared
  `src/search/repo_search/` execution for both plain Julia and plain Modelica
  plugin repositories. The focused host proofs
  `search_intent_routes_code_search_to_plain_julia_plugin_repository` and
  `search_intent_routes_code_search_to_plain_modelica_plugin_repository` are
  green against the linked native parser-summary services after the
  `WendaoSearch.jl` gRPC bootstrap fix.
